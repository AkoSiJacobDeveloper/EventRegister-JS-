In this activity, CRUD stands for Create, Read, Update, and Delete, which are the four basic operations we can perform on data. Hereâ€™s how each part applies to this simple event registration:

    Create: Adding a new event to the list. This is done with the addEventBtn, where I prompt the user for event details and then save it.

    Read: Displaying the list of events. This is handled in the renderEvents function, which reads from localStorage and populates the table with the existing events.

    Update: Modifying an existing event. This is done in the updateEvent function, which prompts the user for new values and updates the selected event in the list.

    Delete: Removing an event from the list. This is handled by the deleteEvent function, which removes the event at the specified index from the list and updates localStorage.
